## 第一个程序

在 JavaScript 中，我们直接就能使用数字，打开浏览器控制台或者文本编辑器输入下边代码并运行。

```javascript
123;
```

你会发现什么都没发生，那是因为，程序在运行，仅此而已。如果我们希望在控制台里输出些什么，可以使用 `console.log`，将代码改成下列这样，你就能在控制台中看到数字`123`了。

```javascript
console.log(123);
```

`console.log`的作用是在控制台输出一些东西，不管是本书还是往后的开发中，都会经常使用到它，现在我们以一个经典的例子结束本节，下面代码就是在控制台输出经典的`hello world!`。

```javascript
console.log('hello world!');
```
> 为什么`hello world!`会用引号包裹着，这个问题我们在字符串一章中会学到，现在可以暂时不管。

## 注释

**注释**，就是给人类阅读的，用来说明某段代码是干什么的说明文字。 JavaScript 程序在执行时会无视注释，注释可以是任何的内容，不管中文、英文、或代码。 JavaScript 提供了两种类型的注释，`//`单行注释和`/**/`多行注释：

```javascript
/*
我是多行注释
JavaScript 程序会无视我的存在
下边代码的作用是在控制台输出 123
*/

console.log(123); // 我是单行注释， JavaScript 程序会无视我的存在，这行代码的作用是在控制台输出 123
```
> 适当的给代码加上注释是个好习惯

## 数字类型的数据及其四则运算

现在来说说我们学习的第一种数据类型**数字**，在 JavaScript 中数字是一种基本的数据类型，我们可以对数字进行一系列的数学运算。

### 数字的加和减

上面简单的使用了数字`123`，JavaScript 中，我们可以像使用计算器一样对数字类型的数据进行算术运算，下边代码中就对数字执行了加减操作。

```javascript
1 + 1; // 得到 2
2 - 1; // 得到 1

1 + 1 + 2； // 得到 4
2 - 1 - 1;  // 得到 0

1 + -1; // 得到 0，这里的 -1 是负一

10 + 5 - 3; // 得到 12
20 - 10 + -5; // 得到 5

```
> 如果你是用文本编辑器编写并运行的上边代码，你会看不到计算的结果，此时就可以使用上文提到的`console.log`将结果输出到控制台，例如，上边的代码可以这么写：
> 
> ```javascript
> console.log(1 + 1); // 在控制台中输出 2
> ```
> 往后的程序中如果有你想在控制台输出的东西，就可以使用`console.log`。

### 数字的乘和除
同时我们也可以对数字执行乘除操作，但是在包括 JavaScript 中，乘和除的写法跟数学中不一样，乘是`*`而除是`/`，下边代码对数字执行了乘除操作：

```javascript
2 * 5; // 得到 10
2 * -1; // 得到 -2

1 * 2 * 3; // 得到 6
2 * 1 * -1;  // 得到 -2

6 / 2; // 得到 3
16 / 2 / 2; // 得到 4

6 * 2 / 2; // 得到 6
6 / 2 * 3; // 得到 9
```

### 括号和优先级

在我们对数字进行算术运算的时候，如果我们混合使用了加减和乘除，那么 JavaScript 会先计算乘除再计算加减例如，这跟数学中的算术运算的规则是一致的：

```javascript
5 + 5 * 2; // 得到 15 而不是 20

10 - 6 ／ 2; // 得到 7 而不是 2

```

这种情况我们称为乘除操作的优先级高于加减操作，优先级高的操作先计算结果，优先级同样的操作从左到右计算，例如：

```javascript
5 + 5 + -10 * 3 / 2 - 2 * 4 + 2; // 得到 -11
```
> 上边的代码按照如下顺序运算首先`-10 * 3 / 2`和`2 * 4`的优先级高  
> 
> 1. 于是从左到右(即先计算`-10 * 3`再将结果除以`2`)计算`-10 * 3 / 2`得到`-15`。
> 2. 计算`2 * 4`得到`6`。
> 
> 于是原式就成了这样：`5 + 5 + -15 - 6`。
> 
> 1. 再从左到右计算上式得到 `-11`。

有的时候我们希望某些部分可以优先计算，我们就可以像数学一样，使用括号`()`来改变优先级，例如：

```javascript
5 + ( 5 + -10)  * 3 / 2 - 2 * (4 + 2); // 得到 -14.5
```
> 上边的代码按照如下顺序运算首先`5 + -10`和`4 + 2`的优先级高  
> 
> 1. 于是从计算`5 + -10`得到`-5`。
> 2. 计算`4 + 2`得到`6`。
> 
> 于是原式就成了这样：`5 + -5 * 3 / 2 - 2 * 6`，因为`-5 * 3 / 2`和`2 * 6`的优先级高。
> 
> 1. 先从左到右计算`-5 * 3 / 2`得到 `-7.5`。
> 2. 再计算 `2 * 6` 得到 `12`。
> 
> 于是原式就成了这样：`5 + -7.5 - 12`，从左到右计算计算结果，就得到了`-14.5`。

## 操作符和操作数

学到这里，其实我们已经接触到了编程语言中两个最基本的概念了，它们分别是操作符（也叫运算符）和操作数。操作符可以对操作数进行相应的操作，例如我们之前学的`+`、`-`、`*`、`/`就属于操作符，在 JavaScript 中它们被称为算数操作符，可以用来进行算术运算。而在它们两端的就叫操作数，例如之前例子使用到的各个数字。例如下边代码中的`+`和`*`就是操作符，`3`、`5`、`10`就是操作数：

```javascript
3 + 5 * 10；
```

小练习：你能分辨出下面代码中哪些是操作符，哪些是操作数么？

```javascript
5 + 5 + -10 * 3 / 2 - 2 * 4 + 2;
```
> 提示： 一共有 8 个操作符，8 个操作数。

我们在往后的学习过程中会发现，操作数可以是数字，也可以是往后学到的各种数据类型，同时，操作数也不一定在操作符的两端，有的操作符只需要一个操作数，而有的需要三个操作数。

## 取模操作符

JavaScript 还可以使用 `%` 运算符对数值类型的数据进行取模（取余数）运算，我们知道在数学中 $$ 3 \div 2 $$ 的余数是 $$ 1 $$，$$ 23 \div 7 $$ 的余数是 $$ 2 $$，$$ 10 \div 3 $$ 的余数是 $$ 1 $$ ，而 $$ 3 \div 3 $$ 的余数是 $$ 0 $$，所以在 JavaScript 中我们有：

``` javascript
  3 % 2; // 得到 1
  23 % 7; // 得到 2
  10 % 3; // 得到 1
  3 % 3; // 得到 0
```

`%` 运算符的优先级跟 `*` `/` 运算符一致，在数学中，乘法和除法的运算优先级总是高于加法和减法，例如 $$ 1 + 3 \times 3 = 1 + ( 3 \times 3 ) = 10 $$ 。而取模运算有着和乘法除法同样的优先级，所以在 JavaScript 中我们有：

```javascript
  // % 的优先级高于 + 和 - 因此：
  
  1 + 3 % 2;
  // 等同于
  1 + ( 3 % 2 );
  
  1 - 3 % 2;
  // 等同于
  1 - ( 3 % 2 );
```

```javascript
  // % 的优先级与 * 和 / 相同，因此：
  
  1 * 4 % 3;
  // 等同于
  ( 1 * 4 ) % 3;
  
  10 / 2 % 3;
  // 等同于
  ( 10 / 2 ) % 3;
  
 
  9 % 2 * 3;
  // 等同于
  ( 9 % 2 ) * 3;
  
  9 % 2 / 1;
  // 等同于
  ( 9 % 2 ) / 1;
```

## 计算机处理数值类型时的误差
现实中，经常会遇到圆周率 $$ \pi $$ ，无限小数$$ \frac{1}{3} $$ 以及自然对数的底数 $$ e $$ 等由无限多位数构成的数。而计算机的内存是有限的，没办法在内存中存放无穷多位数，因而计算机在处理数值时候的精度是有限的。所以包括 JavaScript 在内的许多编程语言在处理小数的时候会有一些误差，比如：

```javascript
  0.1 + 0.2; // 输出 0.30000000000000004 而不是 0.3
  3.1 % 3; // 输出 0.10000000000000009 而不是 0.1
  1 / 3; // 输出 0.3333333333333333 而不是 0.3333......
```

出现这些误差是正常现象，这跟计算机内部处理数值时的算法有关，在做练习或者实际开发中遇到的时候大可不必惊讶，同时这些误差在绝大多数实际开发场景中可以完全忽略。
